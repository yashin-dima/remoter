# Безопасность

## Как устроено доверие

Remoter запускает Claude Code **у вас на Mac** и ходит к проекту по вашему обычному `ssh`.
Ключевые границы:

- **На сервер ничего не ставится.** Всё делается через системный `ssh`/`git`; ваш
  `~/.ssh/config` используется как есть и не изменяется.
- **Настройки и журналы Claude Code читаются только на чтение.** Единственное, что приложение
  пишет, — хуки уведомлений в `.claude/settings.local.json` (свои записи находятся по имени
  скрипта, чужие не трогаются). Этот файл в `.gitignore` и не должен попадать в репозиторий.
- **Локальный редактор Monaco** отдаётся по `http://127.0.0.1` только на loopback, за случайным
  токеном в пути; содержимое ваших файлов через HTTP не отдаётся.
- **Всё, что уходит в шелл, экранируется** (`shq`/`shqPath`), разрушающие команды — с `--`.
- Monaco при сборке сверяется с закреплённым SHA-256 (`Scripts/vendor-monaco.sh`).

## Режим разрешений Claude

По умолчанию приложение **не навязывает** Claude режим разрешений — он берёт свой из
`~/.claude/settings.json`. «Полный доступ» (bypassPermissions, без единого запроса) включается
только осознанно. Учтите это, прежде чем выбирать его: сессия автостартует при открытии проекта.

## Как сообщить об уязвимости

Откройте приватный security advisory на GitHub (**Security → Report a vulnerability**) или
напишите мейнтейнеру. Пожалуйста, не открывайте публичный issue для уязвимостей до того, как
выйдет исправление.
