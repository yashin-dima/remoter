# Remoter — план реализации

> **Статус: реализовано.** Как пользоваться и как всё устроено — в [README.md](README.md).
> Этот файл оставлен как исходный замысел; ниже отмечено, где реальность разошлась с планом.

Нативное Mac-приложение для работы с удалёнными проектами по SSH: дерево файлов, просмотр кода,
наглядный git-diff (side-by-side, зелёный/красный) и встроенный терминал для Claude.
Ничего не устанавливается на сервер — всё работает через обычные SSH-команды.

## Что пришлось изменить по ходу

- **Monaco раздаётся с локального http-сервера, а не с `file://`.** Из `file://` браузер
  запрещает web worker'ы, а Monaco считает diff именно в воркере — зелёного и красного
  не появлялось бы вообще. Это оказалось самой дорогой ошибкой: редактор при этом выглядит
  полностью рабочим.
- **Поллинга `git status` недостаточно.** Когда агент правит файл, который git и так считает
  изменённым, вывод `git status` не меняется. У открытого файла дополнительно сверяется
  контрольная сумма — без этого «живого» обновления diff'а не получается.
- **Добавлены порт и произвольные опции ssh** в настройках проекта: не у всех серверов 22-й порт
  и не для всех есть алиас в `~/.ssh/config`.
- **Терминал — SwiftTerm**, как и планировалось, но подключается через тот же мультиплексор,
  что и команды: открывается мгновенно и не спрашивает пароль второй раз.
- **«Хэш вывода status» нельзя брать из `hashValue`.** На Darwin `Data.hash(into:)` хэширует
  только начало буфера, а `--porcelain=v2 --branch` начинается с oid коммита и имени ветки —
  они меняются лишь при коммите. Правки рабочей копии давали тот же отпечаток, и живое обновление
  молча стояло. Отпечаток — SHA-256 от всего вывода.
- **Обрыв канала надо отличать от ответа команды.** ssh отдаёт на это свой код 255. Пока его
  не разбирали, умерший канал выглядел как «файла нет в ревизии»: изменённый файл рисовался
  целиком добавленным, а сохранение, не сумев сверить чужие правки, писало вслепую.
- **Мультиплексор мало поднять — его надо гасить.** Подобранный чужой (осиротевший) мастер
  не наблюдался и не закрывался: `ssh -O exit` теперь шлётся и ему, иначе фоновый `ssh -N`
  жил вечно, а UI навсегда оставался «на связи».
- **Запись в stdin умершего ssh убивала приложение целиком** — SIGPIPE. Сигнал погашен,
  ошибка записи обрабатывается как ошибка.

## Стек

- **Оболочка:** SwiftUI (macOS 14+). Окно = `NavigationSplitView`: слева дерево файлов и список
  изменений, в центре Monaco, снизу — панель терминала.
- **Рендер кода и diff:** Monaco Editor (движок VS Code) внутри `WKWebView`. Monaco бандлится
  локально в ресурсы приложения (без CDN, работает офлайн). Diff — `monaco.editor.createDiffEditor`,
  это ровно тот самый side-by-side вид «пополам зелёный/красный» с подсветкой синтаксиса.
  Мост Swift ⇄ JS через `WKScriptMessageHandler` / `evaluateJavaScript`.
- **Терминал:** [SwiftTerm](https://github.com/migueldeicaza/SwiftTerm) (нативный эмулятор
  терминала на Swift), в нём запускается `ssh -t <host>` — там живёт Claude и вообще любая
  интерактивная сессия.

## Ключевое архитектурное решение: SSH через ControlMaster

Не реализуем SSH-протокол в приложении (библиотеки на Swift сырые, а auth-зоопарк —
ключи, agent, ProxyJump, 2FA — бесконечен). Вместо этого используем системный `ssh`
с мультиплексированием:

```bash
# одно мастер-соединение на воркспейс (держится в фоне)
ssh -M -S ~/Library/Application Support/Remoter/sockets/<host>.sock -o ControlPersist=yes -N <host>

# все последующие команды летят по уже открытому каналу — без handshake, ~мгновенно
ssh -S <sock> <host> git -C /srv/project status --porcelain=v2 -z
ssh -S <sock> <host> cat /srv/project/src/main.py
```

Плюсы: наследуется весь `~/.ssh/config` пользователя (алиасы, ключи, ProxyJump, agent),
ноль зависимостей, и терминал (`ssh -t -S sock`) подключается через тот же канал мгновенно.

Запись файла при сохранении: `ssh -S sock host 'cat > /path/file.tmp && mv /path/file.tmp /path/file'`
с передачей содержимого через stdin (атомарно через tmp+mv).

## Модель данных

- **Workspace** = `{имя, host (алиас из ssh config), удалённый путь проекта}`. Список воркспейсов
  хранится в Application Support, каждый открывается в своём окне.
- **Дерево файлов:** ленивое — содержимое директории запрашивается при раскрытии
  (`ls -la --` / `find -maxdepth 1`), чтобы не тормозить на больших репозиториях.
  Бейджи статуса (M/A/U) — из `git status`.
- **Список изменений:** парсинг `git status --porcelain=v2 -z` (машинный формат, устойчив
  к пробелам/юникоду в именах).
- **Diff файла:** original = `git show HEAD:<path>` (или `:<path>` для staged-режима),
  modified = текущее содержимое рабочей копии. Оба отдаются в Monaco DiffEditor.
  Untracked-файл — original пустой, весь файл зелёный.

## Обновление (поллинг)

Каждые 2–3 секунды по мастер-каналу: `git status --porcelain=v2 -z`. Сравниваем хэш вывода
с предыдущим — если не изменился, ничего не трогаем (ноль перерисовок). Если изменился:
обновляем список изменений и бейджи дерева; если открытый в diff файл затронут — перечитываем
и обновляем содержимое (Monaco умеет обновлять модель без потери позиции скролла).
Поллинг ставится на паузу, когда окно неактивно.

## Этапы

### M1 — каркас и SSH-слой
- SwiftUI-приложение, экран списка воркспейсов (добавить: host + путь), открытие окна воркспейса.
- `SSHConnection`: запуск/контроль мастер-соединения, `run(command) async -> (stdout, stderr, code)`,
  обработка обрыва (баннер «reconnecting…», авто-переподключение).
- Проверка: выполняется `git status` на реальном сервере, вывод виден в логе.

### M2 — дерево файлов
- Ленивая загрузка директорий, иконки по типам, сортировка (папки выше).
- Бейджи git-статуса на файлах и папках (агрегация вверх по дереву).
- Учет `.gitignore`-подобного скрытия (`.git`, `node_modules` — сворачивать, не грузить заранее).

### M3 — просмотр файла (Monaco)
- Бандл Monaco в ресурсы, обёртка `MonacoView: NSViewRepresentable`.
- Клик по файлу → `cat` по SSH → Monaco с подсветкой по расширению.
- Лимит размера (например, 2 МБ) и детект бинарников (`grep -q $'\0'` / `git diff --numstat`) —
  для них плейсхолдер вместо просмотра.
- Тема Monaco синхронизируется со светлой/тёмной темой macOS.

### M4 — diff (ядро продукта)
- Секция «Changes» в сайдбаре из `git status`.
- Клик по изменённому файлу → DiffEditor: HEAD vs рабочая копия, переключатель
  side-by-side / inline, счётчик +N −M.
- Untracked / deleted / renamed файлы обрабатываются корректно.
- Переключатель «vs HEAD / vs staged» (`git show :path`).

### M5 — поллинг и живое обновление
- Таймер + хэш вывода status, обновление без мерцания, пауза в фоне.
- Обновление открытого diff при изменении файла Claude'ом на сервере — «эффект живого кода».

### M6 — редактирование
- Правая сторона DiffEditor (и обычный просмотр) редактируемые; ⌘S → атомарная запись по SSH.
- Защита от конфликта: перед записью сверить хэш серверной версии с тем, что читали;
  при расхождении — предупреждение (Claude мог изменить файл параллельно).
- Индикатор несохранённых изменений (точка на табе).

### M7 — терминал
- SwiftTerm-панель снизу (сворачиваемая), запускает `ssh -t -S <sock> <host>` сразу в
  `cd <путь проекта>`. Здесь запускается Claude — всё в одном окне.
- Несколько вкладок терминала на воркспейс.

### M8 — полировка
- ⌘P quick-open (fuzzy-поиск по `git ls-files`), ⌘F поиск в файле (встроен в Monaco).
- Табы открытых файлов, восстановление сессии при перезапуске.
- Стабильная подпись самоподписанным сертификатом с первого дня (adhoc-подпись
  сбрасывает TCC-разрешения при каждом ребилде).

## Риски и решения

| Риск | Решение |
|---|---|
| Обрыв SSH / сон Mac | ControlPersist + health-check (`ssh -O check`), авто-reconnect, offline-баннер |
| Огромные репозитории | Ленивое дерево, `--porcelain -z`, лимиты на размер файла |
| Бинарные файлы | Детект и плейсхолдер |
| Не-UTF8 кодировки | Пробуем UTF-8, fallback latin1 + предупреждение |
| Одновременное редактирование с Claude | Сверка хэша перед записью (M6) |
| Проект без git | Дерево и просмотр работают, секция Changes скрыта |
