# Как участвовать

Спасибо за интерес к Remoter! Несколько ориентиров, чтобы правки заходили гладко.

## Сборка и запуск

```sh
./build.sh            # собрать Remoter.app (Monaco подтянется сам, нужны Node/npm и сеть)
open Remoter.app
```

Нужны **Xcode 15+ / Swift 5.9+**. Единственная Swift-зависимость — SwiftTerm, её тянет SwiftPM.

## Тесты

Тесты интеграционные — против настоящего sshd и git. Без стенда половина пропускается, поэтому
запускайте с ним:

```sh
./Tests/local-sshd.sh        # поднимет sshd на 127.0.0.1:2222 и напечатает команду для swift test
# ...скопируйте и выполните напечатанную команду `swift test`...
./Tests/local-sshd.sh stop   # убрать за собой
```

`swift test` без переменных из стенда ничего не доказывает (и `TestIsolation` уведёт данные в
песочницу — на настоящие проекты прогон не попадёт). После правок веб-половины прогоняйте
`./Remoter.app/Contents/MacOS/Remoter --selfcheck`.

## Стиль

- Читайте [CLAUDE.md](CLAUDE.md) — там короткая карта кода и правила (экранирование шелла,
  «только чтение» для настроек Claude, единый `Connection`).
- Цвета, отступы и шрифты в UI — **только** через `Design.swift` (`Theme`, `D.s(...)`).
- Любая строка, уходящая в шелл, — через `shq`/`shqPath`; разрушающие команды — с `--`.
- Новую логику по возможности покрывайте тестом; если она про стыки с ssh/git — интеграционным.

## Pull requests

Небольшие сфокусированные PR проще ревьюить. В описании — что и зачем; если меняется поведение,
скажите, какие тесты это подтверждают.
