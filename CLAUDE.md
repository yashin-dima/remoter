# Remoter

Mac-приложение (SwiftUI, `swift-tools-version:5.9`, macOS 14+): дерево файлов, git-diff,
редактор и терминал для проекта — на сервере (по ssh) или на этом же Mac. Задумано под работу
с Claude Code, который правит код, а пользователь видит правки в diff'е.

## Сборка и запуск

```sh
./build.sh                              # собрать Remoter.app (Monaco подтянется сам через npm)
./Remoter.app/Contents/MacOS/Remoter --selfcheck   # проверить веб-половину (Monaco + diff)
```

Нужны Xcode 15+/Swift 5.9+ и Node.js с npm (только для первой загрузки Monaco).

## Тесты

Интеграционные, против настоящего sshd и git — мокать нечего. Без стенда половина тестов
пропускается, поэтому запускать только с переменными окружения, которые печатает стенд:

```sh
./Tests/local-sshd.sh        # поднимет sshd на 127.0.0.1:2222 и напечатает команду для swift test
./Tests/local-sshd.sh stop   # убрать за собой
```

**Никогда** не запускай `swift test` без этих переменных против настоящих данных: `REMOTER_STORE`
(список проектов), `REMOTER_HOME` (корень локальных папок), `REMOTER_SOCKETS` (ssh-сокеты),
`CLAUDE_CONFIG_DIR` (журналы Claude). Под тестами `TestIsolation` уводит незаданные пути в
`/tmp` сам, но полагаться на это вместо переменных — не стоит.

## Как устроен код

- `Sources/Remoter/Core/` — всё, что общается с внешним миром. Ключевой тип — `Connection`:
  `SSHConnection` (ssh-мультиплексор) и `LocalConnection` (`/bin/sh`). Это **единственное** место,
  где «проект на сервере» отличается от «проекта на этом Mac»; дерево, git, diff и запись работают
  одинаково поверх `Connection.sh`.
- `Sources/Remoter/UI/` — SwiftUI. Цвета, отступы и шрифты — **только** через `Design.swift`
  (`Theme` и `D.s(...)`), чтобы масштаб интерфейса и тёмная тема не разъезжались.
- `WorkspaceModel` разложена по `WorkspaceModel+*.swift` по обязанностям; хранимые свойства —
  в основном файле.

## Правила

- Любая строка, уходящая в шелл, экранируется через `shq` (или `shqPath` для пользовательских
  путей). Разрушающие команды — с `--`. Это дисциплина, а не перестраховка: пути и имена файлов
  приходят с сервера и от пользователя.
- Настройки и журналы Claude Code (`~/.claude`) — **только чтение** (`ClaudeConfig`,
  `ClaudeSessions`, `ClaudeLive`). Единственное исключение — хуки уведомлений в
  `settings.local.json` (свои записи находятся по имени скрипта, чужие не трогаются).
- Формат журналов Claude (`*.jsonl`) чужой и может меняться между версиями: разбор нарочно
  нетребовательный — не нашли поле, показываем что нашли, не падаем.
